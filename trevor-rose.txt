    1. What is Semantic HTML?
    Semantic HTML is HTML code that is used to describe where it is, rather than code that is not semantic and does not describe its location.

    2. What is HTML used for? 
    HTML is used to create a website's structure.

    3. What is an attribute and where do we put it? 
    An attribute is a piece of additional information for the HTML and it goes in the opening tag.

    4. What is the h1 tag used for? How many times should I use it on a page?
    H1 is for the most important title on a page and should only be used once.

    5. Name two tags that have required attributes
    <a (href)> and <img (src)>

    6. What do we put in the head of our HTML document? 
    Typically the main title and navigation are included in the head of the document

    7. What is an id? 
    An id is an identifier for a specific object that is used only once on each page.

    8. What elements can I add an id to? 
    You can add an ID to any element.

    9. How many times can I use the same id on a page? 
    Once.

    10. What is a class? 
    A class is an identifier that is typically used for styling as it can be reused.

    11. What elements can I add a class to? 
    A class can be added to any element.

    12. How many times can I use the same class on a page? 
    A class can be used as many times as needed on any given page.

    13. How do I get my link to open in a new tab?
    <a href="" target="_blank"></a>

    14. What is the alt attribute in the image tag used for? 
    The alt attribute is used in case the image cannot load or for users that are unable to see the image.

    37. Walk me through Lambda's git flow. 

    In the start of a project, you will git clone it to your machine, and checkout to a new branch.  When you are ready to commit, you will add the files in the project, make a commit message, and push the local changes to the remote server, where pull requests and merges can be done.
